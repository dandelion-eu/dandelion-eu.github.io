Dandelion API v1.0

Read API

The data can be queried programmatically through our API. Feel free to experiment the current features and report your user experience through our Feedback system. The awesome Socrata API has been inspiring our design, so that there's an almost total compatibility between the two. Stay tuned - more functionality is on its way!

API endpoint

/api/​v1/​datagem/​{datagemId|datagemSlug}/​data.{outputFormat}

You can obtain the datagem ID by clicking on the "Get API" button on the datagem page.

Output format

Format Description
json An array that contains N members according to the number of results retrieved by the query. The number of results is available as a X-DL-Count header in the response. Each member is a JSON object containing all key-values pairs corresponding to all columns present in the datagem
api Pretty interface for using the API, the full response, comprehensive of headers, is displayed in a dedicated styled page

Parameters

$select
[DISTINCT] {field} [AS "string"] | {function}([DISTINCT] {field} | {value}, ...) [AS "string"], ...

Only shows the specified column, or list of columns separated by comma. Distinct or field functions can be used. It is also possible to rename columns with the AS keyword.

  • /data.api?$select=name,%20category
$where
{field} {binary_op} {value} | {field} {binary_op} {field} | {function}([{param1}, {param2}, ...])

This allows to filter results on one or more columns in a SQL-like language. Allowed operators are < (less than), > (more than), = (equals), != (not equals), =< (equals or less than), => (equals or more than). Filter conditions can be chained with AND and OR operators or negated using NOT. You can use also some filter functions.

The special columns "category" in EuroGeo and POI datasets: they contain the label of the category assigned to each event or location, according to a taxonomy/classification is inspired by Geonames, Schema.org and other sources
  • /data.api?$select=name,%20category

Field functions

Note that field functions produce new columns with their result. The column name is arbitrary set by the API but it is possible to set a custom name by using the AS keyword
Field functions Description
count([column]) Aggregation function for counting (similar to SQL COUNT()). If no parameter is specified the operation is performed on all columns. It is possible to use DISTINCT to consider only different elements.
sum(column) Aggregation function for summing (similar to SQL SUM()). It is possible to use DISTINCT to sum only distinct values. Note: it works only for numeric values.
avg(column) Aggregation function for calculating the average value (similar to SQL AVG()). It is possible to use DISTINCT to consider only distinct values. Note: it works only for numeric values.
min(column) Aggregation function for extracting the minimum value (similar to SQL MIN()).
max(column) Aggregation function for extracting the maximum value (similar to SQL MAX()).
